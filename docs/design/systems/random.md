# 随机性

CardGameDemo是一个rougelike游戏，因此其中存在许多随机事件，需要设计许多随机算法，其中许多随机事件使用类似的随机算法，在本文档中进行论述。



### 伪随机本质的随机性

CardGameDemo使用种子生成对局，并允许在对局过程中保存退出，为了保证使用相同种子时对局中的随机事件相同，所以CardGameDemo中的随机事件本质是种子控制的伪随机事件，在相同的条件下，发生的事件应该相同。



### 骰子事件

CardGameDemo中使用类似DND中使用骰子来控制事件发生的算法，其形式是为可能发生的事件制定一个”点数要求“，当掷出点数不低于点数要求时，判定为事件会发生。

使用的骰子是20面骰。

除了掷出点数不低于点数要求的判定标准之外，还有两种特殊情况：

1. 如果掷出1点，不论分数要求是多少直接判定为失败，事件不会发生。
2. 如果掷出20点，不论分数要求是多少，直接判定为成功，事件会发生。

除此之外，骰子事件还有“劣势”“均势”“优势”之分。

“均势”时，使用一个20面骰掷出点数。

“劣势”时，使用两个20面骰掷出点数，并取点数较小者为最终参与判定的点数。

“优势”时，使用两个20面骰掷出点数，并取点数较大者为最终参与判定的点数。

计算点数时，除了只计算掷出的点数，还有可能涉及其他要素的点数奖励或惩罚，例如假设当前检定的是一个能否躲过陷阱的事件，那么可能会把”敏捷“作为判定因素，会使用一个特殊的敏捷属性补正来结合敏捷属性值计算出奖励或惩罚点数。
