# 游戏事件

本文档中讨论CardGameDemo中可能存在的”游戏事件“，此处的游戏事件和gameplay-framework.md中提到的GameplayEvent并不是同一个概念。

GameplayEvent是游戏实现层面泛用于游戏状态变更传播的信息，而游戏事件指的是被专门设计用于实现各种游戏体验的游戏中可能发生的各种事情。

以下是一些可能的游戏事件的例子：

- 玩家在地牢中触发陷阱，需要进行敏捷判定来判断是否遭到损害。
- 玩家发现了房间中的机关，触发机关可能会发生别的事情。触发机关可能会放出新的怪物，或者开启隐藏房间等等。

可以看出，”游戏事件“本质上提供给玩家某种交互机会，它本质上是游戏设计中一个巨大的宽泛的概念而不一定是一个“类型”。

在CardGameDemo中，游戏事件本质在于它的交互性。一个游戏事件会给出事件的描述，给予一些选项供玩家交互，关键在于“给予玩家选项交互”。
