## TL;DR

本文定义的是 **CardGameDemo 的最终能力愿景**，不是首个可玩版本的范围。  
首个可玩版本可以做切片推进，但最终 Demo 需要覆盖规则框架、调试能力、数据工具链与验证闭环。  
其中“控制台模式 + 战斗独立运行”是核心强需求。

## Demo 最终愿景能力

### 1) 游戏核心框架能力

#### Gameplay Framework

支持大量 Attribute 处理、丰富的 GameplayEffect/Modifier/GameplayAbility，并支持 GameplayEvent 发布-订阅。

#### 多运行模式（强需求）

必须支持以下运行形态：

- **FullAdventureMode**：完整一局（局外 + 地牢 + 战斗）
- **BattleOnlyMode**：可直接开一局战斗，不依赖完整冒险流程
- **DebugConsoleMode**：控制台调试模式（命令式注入与状态查看）

#### 模块可独立运行（强需求）

至少应支持独立运行/测试：

- 战斗系统（回合循环、出牌、结算）
- Gameplay Framework 核心（GE/GA/Event）
- 属性计算管线（含 Stage）

#### 可复现与可观测

- 固定 seed 可复现同一流程
- 输出结构化日志（事件、状态变化、结算过程）
- 支持基础回放（至少重放一场 BattleOnly）

### 2) 数据驱动与工具链能力

#### GameplayTag 编辑与导入

支持多来源 GameplayTag 导入与易用编辑（例如表格导入/导出）。

#### 卡牌编辑器

支持修改卡牌效果与关键参数，并可做基础引用校验。

#### 装备编辑器

支持修改装备加成数值、装备效果、装备提供卡牌，并可做基础引用校验。

#### 数据校验工具

至少支持：

- Tag 合法性与层级校验
- 卡牌/装备/角色引用完整性校验
- 核心参数范围校验（类型、上下限、枚举）

### 3) 调试与验证能力

#### 控制台调试能力

支持通过控制台命令快速构造和验证场景，例如：

- 直接创建战斗并注入角色、装备、手牌、状态
- 查看当前关键状态快照
- 触发指定事件或执行一步操作

#### 自动化验证基础

支持批量自动战斗模拟与固定输入回归场景，用于验证数值调整与规则变更。

### 4) AI 友好能力

支持将系统状态与合法动作程序化暴露，便于后续 AI 自动游玩与策略验证：

- 可枚举合法行动
- 可读取结构化状态
- 可提交行动并获取新状态

## Demo 之后建议补充

### 1) 更完善的回放与诊断

- 全流程日志归档
- 关键结算步骤可追踪
- 异常自动标注与最小复现导出

### 2) 更深入的数据编辑体验

- 更复杂效果的可视化配置
- 规则编辑器与数据校验联动

### 3) 规则与内容进一步解耦

- 新卡牌/装备尽量不改核心代码
- 高复杂效果通过可扩展计算钩子处理
