#### 如何理解”效果“

现在考虑对于不同的”效果“的理解，广义上讲，”效果“指的是**装备、卡牌或道具**产生的任何影响，例如造成伤害，获得格挡，抽牌、弃牌、使玩家获得”每当你抽1张牌，你对一个随机敌人造成若干伤害“这种”能力“，使某个单位被赋予”受到的伤害+25%“，”受到的伤害-1“。这些都是广义上的”效果“。

而狭义上的”效果“，一般指的是施加的Buff/Debuff，在此我们也应当厘清一下Buff在语义上的定义，在本项目中，我们把”受到的伤害+25%“，”持续1回合的受到的伤害+25%“，”造成的伤害+1“，”在2回合内造成的伤害+1“这种描述称为Buff（对受用者而言是正面的）/Debuff（负面）。必须强调的是，这只是为了在本项目内讨论的“用词”统一而不是一种实现上的概念划分，实现上我们不一定真的要做一个类称为“Buff”或“Debuff”来一对一实现这个概念，这只是为了我们在本项目中讨论的方便。

而对于像”每当你抽1张牌，你对一个随机敌人造成若干伤害“这种描述，我们在本项目中称其为“能力”（Ability），它们的特点是往往实现了一种特别的“规则”，例如上面这个能力就是添加了一条规则，触发时机为玩家抽牌时，执行效果为“对一个随机敌人造成若干伤害”。

事实上，广义上来讲Buff/Debuff也可以视作为一种规则，例如把角色受到的伤害增加25%可以看作是角色吸收伤害时的规则改变了，但是为了在语言和定义上区分，我们只把那些看起来特别像“规则”的“规则”称之为“规则”。



效果具有“可堆叠”的性质，这指的是部分效果具有可以堆叠的可能性而不是所有效果都可以堆叠，堆叠的方式也不相同。

考虑效果“持续1回合的受到的伤害+1”，如果一个单位在已经拥有一层这个效果的情况下又获得了一层，那么应该如何堆叠？是变成“持续2回合的受到的伤害+1”，还是“持续1回合的受到的伤害+2”？单纯从文字描述上来说这都是合理的。但在实际情况上来讲，应当能从效果的描述可能能看出区别。例如卡牌的描述为“使一个单位在本回合中受到的伤害+1"，那么打出两张这张牌的效果很明显就是”持续1回合的受到的伤害+2“

我们应当明确规范效果的可堆叠性，因此我们应当为每一种效果命名，至少是在概念上我们明确知道如何称呼一种效果，并且规范某种效果的堆叠规则，例如考虑”持续1回合的受到的伤害+25%“，我们可能称之为”易伤“，并且我们规定”易伤“的堆叠规则是堆叠持续时间而不是数值百分比。

我们把可堆叠效果的堆叠数量称之为”层数“，例如在上面的例子中，2层的易伤就是”持续2回合的受到的伤害+25%“。



#### 效果的执行与时序

装备、卡牌、道具的效果的执行时序以及数值计算的顺序是非常重要的。

“时序”的含义非常广，尺度不同，考虑以下时间：

- 地牢关卡尺度：
  - 进入/离开一个房间时
  - 进入/离开一层时
  - 道具相关：
    - 拾取道具时
    - 使用道具时
    - 丢弃道具时
  - 装备相关：
    - 穿戴装备时
    - 脱除装备时
  - 事件尺度
    - 开始/结束事件时
  - 战斗尺度
    - 开始/结束一场战斗时
    - 回合尺度
      - 开始/结束你的/敌人的回合时
      - 回合动作尺度
        - 造成伤害时
        - 受到伤害时
        - 获得格挡时
        - 抽牌时
        - 弃牌时
        - 将弃牌堆洗入抽牌堆时
        - 造成XXX效果时
        - 消耗Cost时
        - 获得Cost时
        - 使用道具时
        - ……

其中战斗尺度的时序尤为重要。

考虑这样的情况：假设打出了一张牌，其效果时“打出你抽牌堆顶的一张牌，之后做一些事情”，那么从描述上看，其执行顺序就应该是先打出这张牌，也就要转入这张牌的执行环节，完成之后才能做后续的事情，类似于一种嵌套调用。

在一些常驻效果生效的情况下，可能牵连多个事件，例如：“每当你对敌人造成伤害，造成伤害增加一定数值”，这可以视作一项增伤计算而不是新事件，计算逻辑参考下文；“每当你对敌人造成伤害，额外造成伤害”，这应该算作触发了新的“造成伤害”事件，效用应额外结算。

考虑卡牌的数值计算，假设现在取属性中性值为10，出牌者力量值为14，出牌者身上有“造成伤害减50%”的效果，对一个具有2点护甲，身上具有“受到伤害+25%（向上取整）”效果的敌人打出一张造成4点物理伤害的力量补正为A的斩击，基本的计算如此：

最终伤害 = ((面板伤害{4}+ (总属性值 - 中性值){14-10} * 100%} * (1 + 出牌者伤害加成{-50%}) * (1 + 目标受伤加成{+25%}) -  护甲{2} =  ((4 + (14-10) * 1.0) * (1 - 0.5) * (1 + 0.25) - 2 = 3

若继续引入效果：

- 卡牌面板+X/按倍率增加，字面意思
- 总属性值+X/按倍率增加，字面意思
- 在计算这张牌的效用时，总属性值+X/按倍率增加，字面意思，仅对本张牌的计算生效
- 伤害修正x1.25，是应该在护甲减伤之前还是之后算？应该是减伤前
- 造成伤害+25%，是应该乘算还是加算？加算，也就是令出牌者伤害加成变成-25%
- 造成伤害+X，应该算在哪里？算在乘积因子之前的加法部分还是算在乘法之后？（吃不吃伤害加成？吃伤害加成——面板加成，不吃——固定加成）
- 造成伤害时额外**产生**/**造成**伤害，产生的额外伤害单独计算
- 固定/百分比护甲穿透，使得参与计算的护甲值减少
- 按特定元素数量增加伤害（例如手牌数量，抽牌堆牌数等），和“造成伤害+X”类似
- 造成穿甲伤害，忽略护甲减伤计算部分
- 减少目标生命值，不算作“伤害”，直接减少数值



在游戏中往往涉及如上的大量数值计算，它们往往从效果出发，也许有不同的计算规则，我们可能称之为“数值计算管线”？
