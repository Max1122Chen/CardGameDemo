# 冒险与地牢

游戏流程可大致分为地牢外和地牢内两个场景。

在地牢外，玩家可以随意选择装备、道具，变卖物品，选择角色，完成进入地牢前的准备。

玩家进入地牢视作开始一局游戏，我们姑且称之为”一场冒险“来避免”一局游戏“这样可能导致歧义的表达。

在一局冒险中，玩家需要在地牢中选择采取各种行动，包括在房间内移动，和NPC进行交互，触发事件、选择进行战斗，整理背包物品，使用道具、休息等等。



#### 地牢结构

地牢分为多个“层级”，每个层级可能有不同的房间、敌人、NPC、事件以及战利品掉落。

地牢的每个层级本质是一张“图”（要不要设计一些非连通区域需要特殊方法到达？），由许多房间节点组成。

层级的大小并不固定，但总体来说有一个设计好的平均值。



玩家通常无法从下层回到上层，只能从上层移动到下层，移动的条件通常是找到层级中的特殊房间**BOSS房**并击败BOSS，或其他特殊手段。一般来说，在可以从上层移动到下层时，也允许玩家选择带着战利品离开地牢，视作一局游戏胜利。



地牢中的最简单的房间结构是一个正方形，上下左右有可能有门连接到其他房间。也可能有其他结构，例如L型，T型，I型（走廊或长房间），O型（更大的正方形）。一般来说，房间的结构以小正方形为主，其他结构为辅。

两个房间的墙壁可以相邻但不相通。



在进入每一层时，玩家所出生的房间称为”初始房间“，必然是一个初始”安全“的房间，指的是玩家刚落地时，这个房间必然不会出现敌人或事件，但有可能会出现战利品或NPC，地牢生成的规则第一条是以初始房间为起点开始随机生成。

BOSS房应当生成得尽可能离初始房间”远“，而不是非常”靠近“初始房间，这里的远近指的是玩家到达BOSS房所需要花费的成本和行动的丰富程度，成本包括路线长度（不是直线距离），道中战损，花费的时间等。初始房间和BOSS房间相邻但不相连，需要绕很多个房间才能到达，经过多场战斗的情况是可以接受的。BOSS房需要离初始房间较远的意义在于保证玩家一层中能够得到一定的挑战和足够的游玩时长，让玩家不至于觉得一层太短但不应该太长。

一个未探索的房间中可能会出现以下情况：

1. 什么都没有（不多）
2. 有一些战利品可以直接搜刮（相对较少）
3. 有单位在内部
   1. 有敌人（很可能）
   2. 有中立单位（很可能，但比敌人出现概率低一些）
   3. 有友好的NPC（可能，商人这类功能性NPC概率比有战斗能力的AI冒险者更大）
4. 有特殊事件（可能）
5. 有关底BOSS

其中2，3.1，3.2，3.3，4有可能同时出现（敌人、中立单位、友好NPC之间可能发生冲突）

为了保证玩家每一层的游戏体验，有一些类型的房间必然会生成1个或多个，例如BOSS房必须保证每层至少有一个（我们需要多个吗？），应当保证至少有一个商店类型的房间，保证至少有一个房间中会出现事件，保证至少有一个房间中会出现敌人，保证至少有一个房间中有直接可搜刮的战利品。



玩家对房间有“视野”的概念，指的是玩家在一个房间内时对与其相邻的其他房间中的情况的信息获取，玩家可能缺少对某些房间的“视野”，导致无法知道另外一个房间内的情况。玩家在一个房间中对与其相邻的其他房间的视野情况取决于玩家所在房间的特性，这也很好理解，想象地牢中的房间有些可能有大门阻断有些可能没有，玩家若是打开一扇大门，那么有可能惊扰房间中的单位，玩家不需要打开大门时，则有机会偷偷观察房间中的情况。

玩家有可能通过其他特殊的方式来获得对相邻的或更远处的房间的暂时性的视野，例如使用特殊道具。



一个层级中的单位数量应当是有限的。

一个层级中有同时容纳敌人和中立单位的上限和下限，我们称其为”存活单位池“，在生成地牢时，会随机产生介于存活单位池上限和下限数量的敌人和中立单位并分布到地牢中。

层级还有”已讨伐单位下限“，用于维护层级中敌人和中立单位动态变化的情况。层级中的敌人和中立单位有可能在回合结束时选择离开层级，其离开层级的意愿由自己决定，单位的离开不会使得敌人和中立单位的数量降到存活单位池的下限。如果玩家目前已讨伐的敌人或中立单位数量没有达到已讨伐单位下限，并且存活单位数量没有到达存活单位池上限，那么在一轮结束时就有概率生成新的单位，这个概率随着已讨伐单位数量逼近已讨伐单位下限和存活单位数量逼近存活单位池上限的程度而降低。

也就是说，地牢会保证一定的动态性的同时，中并不会无限地刷新出怪物。

#### 探索地牢

我们把在地牢中战斗以外的环节称为”探索“环节。

在探索中，玩家以”轮“为行动单位，每一轮中，玩家拥有有限的行动资源来行动，主要的行动资源消耗在于“移动”。

在一轮中，玩家可通过消耗行动力来执行任意次移动直到行动力消耗完，行动力消耗完后玩家不再能执行“移动”但是可以执行包括和NPC交互、触发事件、整理背包物品、使用道具、休息。例如假设玩家当前的行动力为3点，且每一个房间的通行成本都是1点行动力，那么玩家在本轮最多移动3次，最远移动3格，但中途执行的其他行动不消耗行动力。在一“轮”开始时，玩家通常会回复所有的行动力。实际中，不一定每个房间的通行成本都是1点行动力，例如可能有“走廊”这种房间，只消耗0.5行动力，而一些大房间或特殊房间可能需要消耗更多行动力。



“轮”也是地牢状态刷新的单位，每一轮开始/结束时，地牢中的状态可能发生刷新，体现在单位的分布、刷新、事件的持续等等。



玩家选择进行一次休息时，会提供所需时间长度不同的选择，以“轮”为时间单位，休息轮数越长，休息的效果越好。

但同时休息时会面临一些风险，休息有可能被游荡的敌人或敌人的刷新打断，并且强行进入战斗，亦或者是遭遇一些休息时能触发的特殊事件。

玩家选择休息的地点和时机会影响休息时出现的情况，在某些房间中，玩家在休息时遭遇干扰的可能性将大大降低甚至减少至0，例如当玩家与友好且有战斗力的NPC共处一室休息时，遭遇敌人和负面事件的可能性会降低，反过来，玩家与中立单位共处一室休息时，可能性会增加。但是这并不绝对，只是一种设计时的倾向。



##### 战利品掉落

此处的战利品掉落包括与单位战斗后产生的掉落物以及搜刮房间时出现的战利品。

不同的单位和房间战利品刷新点应当有各自的“掉落物池”。
